- **isGameOver() (~42 lines) mixes state checking with printing the full ANSI result screen** (−1)

  A function should either check something or display something, not both. If the win condition changes, you now have to dig through display code to find it. Split into `checkEnd()` and `showResult()`.

- **showStatus() repeats near-identical skill-card blocks for player 1 and player 2** (−0.5)

  When two blocks are almost identical, any bug fix has to be made twice — it's easy to forget one. Extract a helper `showSkillCards(const Hand&)` and call it for each player.

  ```cpp
  case 1: for (Card& c : p1.getskillCards()) cout << c;  // game.cpp:128
  case 2: for (Card& c : p2.getskillCards()) cout << c;  // game.cpp:142 — identical
  ```

- **getskillCards() all lowercase, inconsistent with getSuit()/getRank() camelCase** (−0.5)

  Consistent naming lets readers predict what a function is called without looking it up — mixing styles breaks that expectation.

  ```cpp
  std::vector<Card>& getskillCards();  // hand.hh:27 — should be getSkillCards()
  ```

- **getsuitpattern()/getRankpattern() mix case conventions** (−0.5)

  One starts lowercase, the other uses camelCase mid-word. Pick one style and apply it everywhere.

  ```cpp
  std::string getsuitpattern(const Card&) const;  // card.hh:18
  std::string getRankpattern(const Card&) const;  // card.hh:19
  ```

- **Non-portable include paths (9 warnings — lowercase files, uppercase includes)** (−0.5)

  macOS file systems are case-insensitive so `"Card.hh"` finds `card.hh`, but Linux is case-sensitive and will fail to compile. Always match the include string exactly to the filename on disk.

  ```cpp
  #include "Card.hh"  // hand.hh:5 — file on disk is card.hh (fails on Linux)
  ```

- **Both #ifndef and #pragma once in the same headers** (−0.5)

  Both do the same job: prevent a header from being included twice. Using both is harmless but signals you don't understand what each one does. Pick one — `#pragma once` is simpler.

  ```cpp
  #ifndef HAND_HH  // hand.hh:1
  #define HAND_HH  // hand.hh:2
  #pragma once     // hand.hh:3 — pick one, not both
  ```
